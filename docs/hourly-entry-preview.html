Why this

Trade-off

  • Lowest discoverability of the three
  • Dashboard-only, like Option B

Recommendation

Go with Option A (sidebar item). It's the only one reachable from every page, it reuses a pattern managers already know, and the pulse dot sells the "live" nature. It's also the smallest code change — one nav row + a route case that calls the openHourlySnapshot() we already built. If you want it doubly visible, add Option B too — they don't conflict (sidebar = global, header pill = in-context shortcut). I'd skip C unless you specifically want the lightest footprint.
